# Разработка балансировщика нагрузки для кластерной инфраструктуры

Идея:
```
Создать собственный балансировщик нагрузки (load balancer) для кластера из серверов/рабочих станций, который распределяет задачи (compute jobs) по узлам исходя из метрик CPU/RAM/задержки, а также динамически перераспределяет задачи при появлении новых узлов или изменении нагрузки.
```

# Ключевые компоненты для реализации:
- Агент на узлах для сбора метрик
- Центральный оркестратор или распределённый алгоритм выбора
- Протокол коммуникации (gRPC, WebSockets)
- UI/CLI для управления
