# Разработка балансировщика нагрузки для кластерной инфраструктуры
Идея:
```
Создать собственный балансировщик нагрузки (load balancer) для кластера из
серверов/рабочих станций, который распределяет задачи (compute jobs) по
узлам исходя из метрик CPU/RAM/задержки, а также динамически
перераспределяет задачи при появлении новых узлов или изменении нагрузки.
```

## Ключевые компоненты для реализации
- Агент на узлах для сбора метрик
- Центральный оркестратор или распределённый алгоритм выбора. 
- Протокол коммуникации (gRPC, WebSockets)
- UI/CLI для управления и мониторинга (CPU, GPU, RAM, диски, сеть)

## Основные пункты
### 1 часть. Анализ существующих подходов к балансировке:
- Round-Robin (циклическая)
- Least-Connections (на узел с минимальными активными задачами)
- Weighted algorithms (с учетом производительности узла)
- Self-organized approaches (Balanced Overlay Networks) — распределённая балансировка без центрального контроллера
- Kubernetes scheduler

### 2 часть. Реализация балансировщика
Можно выбрать одну из архитектур:
- Централизованный Scheduler. Управляющий модуль собирает метрики от агентов и принимает решения.
- Децентрализованный Routing. Узлы сами принимают решения, например, по Gossip-протоколам или распределённому хешированию.
Базовая функциональность должна включать:
- Очередь задач
- Распределение и перераспределение задач
- Обработка отказов узлов
- Метрики эффективности балансровщика

### 3 часть. Экспериментальная часть
- Провести эксперименты на кластере и сравнить методы
- Визуализировать результаты
